Moovi (Fall 2021)

Repository

This project was created in my CSC 4330 class (AKA Software Systems Development). I was the Scrum Master for a group of six, and we had one semester to build this project from the group up with a focus on creating an MVP.

The project was made in Android Studio, using Flutter and the Dart Language. I created the database using the Floor API, which uses SQLite. I gained experience in making a professional UI because my wonderful professor, Dr. Nash, pushed us to continually clean up the look and feel.

Moovi is a mobile app to help users minimize the amount of time they spend searching for a movie to watch. With a swipable interface similar to Tinder, users swipe on movies to add them to a watchlist or send them to the back of the line. One of the more important features is the Friends feature, where users can see a list of movies that they and a friend are both interested in watching. The UI is intended for speed because we want users to move quickly through the list of movies.

If we continue this project, we would first implement better filtering so users can see movies in genres and streaming services of their choosing. We would host the database on a server, instead of making the database local to the device, which we did for the purposes of making an MVP for the semester. The database would need more movies and information so we could make an algorithm to pick movies that might interest each user.

DrugHub (Spring 2021)

Repository

This project was created in my CSC 3380 class (AKA Object Oriented Design). I was the Implementation Leader for a group of six, and we had one semester to build this project with a focus on good initial design.

The project was built in Android studio. I created the database (my first) using the Room API, which uses SQLite. I also helped to create an ER Diagram using Enterprise Architecture.

DrugHub is an android application for helping users remember to take their medications. Users could see a list of upcoming medications and have reminders texted to their phone. We created a form where users enter information about the medication and when they would like to be reminded.

Because the focus of the project was on early design, we did not get a chance to implement a clean and intuitive UI, so if we were to complete this project, this would be the first thing to fix. Currently, the information is specific to the user, so keeping the database of the user's information local to the device would work. Since medications are private information, we would need to increase the app's security.

StarLord (2019)

Repository Reveal Video

This was my second and last year as my High School Robotics Team's Programming Lead. I was one of the only ones on the programming team, so I programmed most of the robot.

This was the first year that we implemented Vision Processing. This was a special project that I was very invested in. Vision Processing is when the robot uses a camera and a light to see where certain things are. The game field had special reflective tape around goals, so we used the shapes of the tape to trigger certain automatic actions in the robot. For instance, the driver could get the robot close to the goal area and press a button, allowing the bot to automatically line up to the goal.

At the end of every match, there was a goal to lift your robot onto different leveled boxes. Our robot had completely automated climbing because there were 2 motors that lifted the robot. This meant that a person would struggle to keep the robot level, but I programmed a way for the robot to measure the angle and change motor speed on either side to keep the robot upright.

Bernoulli (2018)

Repository Reveal Video

This was my first year as my High School Robotics Team's Programming Lead. I was one of the only ones on the programming team, so I programmed most of the robot. The robot's name was Bernoulli because we 'Swiss-cheesed' the robot, meaning we cut holes into the metal panelling so that the robot was under the FRC weight limit, and Bernoulli was a Swiss Mathematician.

There is a 15 second period at the beginning of every robotics match for automated controls. I had full control over what the robot was going to do during those 15 seconds. This particular year was interesting because there were these scales that randomized every match, which side was your team's side. We wanted to score goal at the beginning of every match, so I had to make the robot get information and decide where to score.
